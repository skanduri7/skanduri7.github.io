Part 1.1: Finite Difference Operator

In this section, I implemented two convolution functions, one with four for loops and one with two for loops. I also compared performance with scipy's built-in convolution function.

Convolution Implementation

Here's our numpy-only convolution implementation with proper zero-padding:

Implementation Details: The convolution is implemented using nested loops to iterate over the input image and kernel. Padding ensures that the output has the same dimensions as the input, handling boundaries by filling with zeros. 
              
def conv2d_four(img, kernel):
    H, W = img.shape
    h, w = kernel.shape

    conv = np.zeros_like(img, dtype=np.float32)

    pad = np.pad(img, ((h // 2, h // 2),(w // 2, w // 2)), mode='constant')
    kernal_flipped = np.flipud(np.fliplr(kernel))

    for y in range(H):
        for x in range(W):
            temp_conv = 0.0
            for i in range(h):
                for j in range(w):
                    temp_conv += pad[y + i, x + j] * kernal_flipped[i, j]

            conv[y, x] = temp_conv

    return conv

def conv2d_two(img, kernel):
    H, W = img.shape
    h, w = kernel.shape

    conv = np.zeros_like(img)

    pad = np.pad(img, ((h // 2, h // 2),(w // 2, w // 2)), mode='constant')
    kernal_flipped = np.flipud(np.fliplr(kernel))

    for y in range(H):
        for x in range(W):
            conv[y, x] = np.sum(pad[y: y + h, x: x + w] * kernal_flipped)

    return conv

Performance Comparison

Comparing our implementation with scipy.signal.convolve2d:

Runtime Results

  • Four loop implementation: 2.1811 sec, 0.1661 sec, 0.1742 sec, for an average of 0.8405 sec
  • Two loop implementation: 0.2596 sec, 0.2306 sec, 0.2275 sec, for an average of 0.2392 sec
  • scipy.signal.convolve2d: 0.0122 sec, 0.0012 sec, 0.0013 sec, for an average of 0.0049

My implementations of the convolutions were the four look and two loop. On average and when the kernel was not very small the four loop predictably was the slowest. When a smaller kernel was used the four loop moved slightly faster then the two loop. The built-in scipy convolution function was by far the fastest.

Boundary Handling

  • Our implementation: Used zero padding as per instructions, where the image frame was padded with zeros to keep image size consistent.
  • scipy.signal.convolve2d: To keep consistency, used the option to have zero padding
