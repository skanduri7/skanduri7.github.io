Overview

This project explores image warping and mosaicing using homography transformations. I captured multiple sets of overlapping photographs and implemented algorithms to compute homographies from point correspondences, warp images using different interpolation methods, and blend them into panoramic mosaics.

A.2: Recover Homographies

I implemented a function to recover homography matrices from point correspondences. The function sets up a linear system of equations based on the correspondance points and solves it using least squares.

Mathematical Foundation: Given point correspondences (x, y) → (x', y'), we set up the equation system Ah = b where h contains the 8 unknown entries of the homography matrix H. Each point correspondence contributes two equations to the system.

Point Correspondences for 3-Image Set

Using the 3-image room set, I manually selected corresponding features between adjacent pairs. For a 3-image mosaic, we need 2 sets of correspondences: Image1↔Image2 and Image2↔Image3.

Correspondence Points

System of Equations

For the 3-image mosaic, I solved two separate homography systems. Each uses 8 point correspondences, creating a 16 × 9 overdetermined system solved using least squares.

For each correspondence point (x, y) → (x', y'), we get two equations:

x * h₁ + y * h₂ + h₃ - x' * x * h₇ - x' * y * h₈ = x'

x * h₄ + y * h₅ + h₆ - y' * x * h₇ - y' * y * h₈ = y'

A.3: Warp the Images

Interpolation Method Comparison

Comparing nearest neighbor vs bilinear interpolation quality:

Quality vs Speed Trade-offs: Nearest neighbor is faster but produces blocky artifacts. Bilinear interpolation is slower but produces smoother results with better visual quality, especially noticeable in regions with high detail or curved edges.

Nearest Neighbor: This method simply rounds the mapped coordinates to the nearest integer pixel location and uses that pixel's value directly. It is fast but can create jagged edges and blocky artifacts, especially when enlarging images or applying significant transformations.

Bilinear: This method uses the four nearest pixels around the mapped coordinate and computes a weighted average based on the fractional distances to each neighbor. This creates smoother transitions between pixels and reduces aliasing artifacts, but requires more computation due to the interpolation calculations.

B.1: Harris Corner Detection

Harris Corner Detection: The Harris corner detector identifies interest points by analyzing the local structure of intensity gradients. This is our initial set of points of interest that we will filter down.

Adaptive Non-Maximum Suppression (ANMS): After detecting Harris corners, ANMS selects a set of corner points. For each corner point, it computes the distance to the nearest stronger corner which is called the suppression radius. For this project, I made it so AMNS selects the top thousand points with the largest radius, ensuring we have features across the image rather than clustering in one high texture region.

Point Detection

B.2: Feature Descriptor Extraction

Feature Descriptor Extraction: For each detected point, I extract its local image patch of size 40×40 pixels centered at the corner location. This patch is downsampled and normalized to an 8×8 pixel descriptor to focus on low frequencies. The descriptor essentially captures each interest point in a compact representation to search for.

Normalized Feature Descriptors

B.3: Feature Matching

Feature Matching: I compute the distance between all pairs of feature descriptors from the two images. For each feature in the first image, I identify its closest match in the second image based on descriptor similarity. To filter out ambiguous matches, for each feature, I compute the ratio of distances of the two nearest matches. The final match is only accepted if this ratio is below a threshold which I set to 0.6, ensuring that the best match is significantly better than alternatives.

B.4: RANSAC for Robust Homography

RANSAC: RANSAC is used to estimate homography matrices. The algorithm randomly samples 4 points, which is the minimum needed to compute a homography matrix. It then computes a temporary homography matrix and counts how many other point correspondences are consistent with this. To compare the number of point corresopndances that are consistent with this matrix between iterations, I counted the number of corresponding points that are distanced below a certain threshold (3.0 in my case). After sampling 1000 times, it selects the homography with the most consistency.
